# h5-page-guide

## 项目简介

`h5-page-guide` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本项目不针对任何特定域名或网站，仅作为 HTML 页面集合的存储与展示用途。  
每个页面均为独立、自包含的 HTML 文件，可直接在浏览器中打开查看。

## 目录说明

```
h5-page-guide/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的公共资源（图片、样式、脚本等）
├── index.html      # 项目首页（页面导航列表）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件对应一个独立页面，命名以功能或内容主题为基础。
- `assets/`：存放多个页面共用的资源文件，避免重复。
- `index.html`：提供所有页面的索引链接，方便快速浏览。

## 页面归档说明

- 所有页面均为纯前端实现，不依赖后端服务。
- 页面之间相互独立，无数据共享或依赖关系。
- 归档内容以 HTML、CSS、JavaScript 为主，部分页面可能包含外部库（如 CDN 引用）。
- 每个页面在 `pages/` 下以独立文件夹或文件形式存放，便于管理和引用。

## 使用方式

1. 克隆或下载本仓库到本地：
   ```bash
   git clone https://github.com/your-username/h5-page-guide.git
   ```
2. 打开 `index.html` 查看页面列表。
3. 点击列表中的链接，或直接在浏览器中打开 `pages/` 下的 HTML 文件。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面归档，或改进现有页面。
- 提交新页面时，请遵循以下规范：
  - 页面文件放入 `pages/` 目录，建议以功能或主题命名文件夹。
  - 公共资源尽量放入 `assets/` 目录，避免页面间重复。
  - 确保页面自包含，无需额外配置即可在浏览器中直接运行。
- 如需修改或删除页面，请确保不影响其他页面的正常运行。

## 注意事项

- 本仓库仅作技术归档与展示用途，不保证页面内容长期可用。
- 部分页面可能引用了外部资源（如字体、图标库），请确保网络畅通。
- 如有问题或建议，欢迎通过 Issues 反馈。
